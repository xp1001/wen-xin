# 问心 (Wen-Xin)

**完成任务或汇报前，问自己内心：目标真的达成了吗？**

**作者**: XP
**主页**: [github.com/xp1001/wen-xin](https://github.com/xp1001/wen-xin)

问心是一个 OpenCode meta-skill，在每次你即将声称完成、表达满意、或准备阶段性汇报时，自动触发「灵魂发问」式自我审视，发现问题立即修正，不到位不允许收工。

## 核心机制

### 8 条灵魂发问

| # | 问题 | 用意 |
|---|------|------|
| 1 | **目标还原** | 我准确理解了用户最初的完整需求吗？ |
| 2 | **隐含边界** | 用户没说但默认期望的事做了吗？ |
| 3 | **少做了吗** | 有没有跳过觉得"不重要"的部分？ |
| 4 | **多做了吗** | 做了用户没要求的事？（可能是浪费） |
| 5 | **证据在哪** | 有什么具体证据证明目标达成了？ |
| 6 | **用户视角** | 如果我是用户，会满意吗？ |
| 7 | **打开看看** | 真的确认过结果文件的内容？ |
| 8 | **匆忙了吗** | 为了快而放松了标准？ |

### 修正循环

发现问题后不只会输出建议，而是：

```
灵魂发问发现问题 → 当场修复 → 重新验证 → 再次发问
     ↑                                    |
     └──────── 还有问题？继续修正 ────────┘
```

只有全部通过才允许声称完成。

### 三档判定

| 状态 | 含义 |
|------|------|
| 已完成 | 8 条全部无存疑，证据确凿 |
| 基本完成（有瑕疵） | 存在次要问题，但核心目标达成 |
| 未完成 | 核心目标未达成或发现实质性 Bug |

## 触发时机

- **完成前**：要说"完成""搞定""测试通过"之前，先问心
- **汇报前**：要说"先汇报一下""暂时做到这里"之前，先问心

## 安装

```bash
# 克隆到 OpenCode skills 目录
git clone git@github.com:xp1001/wen-xin.git ~/.agents/skills/wen-xin
```

## 验证测试

用 3 个场景验证了问心效果：

| 场景 | 结果 |
|------|------|
| 完整任务（冒泡排序） | ✅ 正确判定「已完成」，8 条全部通过 |
| 遗漏日志（排序无日志） | ✅ 发问发现遗漏 → 自动补日志 → 再验证通过 |
| NaN Bug（去重函数） | ✅ 发问发现 Bug → 自动修复 → 测试通过 |
