# 问心 (Wen-Xin)

**完成任务或汇报前，问自己内心：目标真的达成了吗？**

**作者**: XP
**主页**: [github.com/xp1001/wen-xin](https://github.com/xp1001/wen-xin)

问心是一个 OpenCode meta-skill，在每次你即将声称完成、表达满意、或准备阶段性汇报时，自动触发「灵魂发问」式自我审视，发现问题立即修正，不到位不允许收工。

## 核心机制

### 8 条证伪式核查

不是自问自答的反思，而是**审产出**：

| # | 行动 | 怎么做 | 什么算发现问题 |
|---|------|--------|--------------|
| 1 | **逐条对照** | 需求逐条列出，从输出中找对应证据 | 有需求找不到对应 → 未覆盖 |
| 2 | **触发异常** | 构造异常场景（空数据、断网、格式错），实际跑一次 | 报错或结果不对 → 边界没处理好 |
| 3 | **精确计数** | 预期数量 vs 实际数量，列出差异 | 数字对不上 → 有遗漏 |
| 4 | **必要性标注** | 逐功能标来源：用户要求 vs 自行添加 | 多余项说不清必要性 → 浪费 |
| 5 | **重跑取证** | 验证命令再跑一次，拒绝引用历史输出 | 重跑失败 → 证据不成立 |
| 6 | **抽样展示** | 从结果中随机取 1 条，逐字段展示 | 字段空/乱码/不对 → 用户不会满意 |
| 7 | **头中尾取样** | 第 1 条、中间 1 条、最后 1 条，逐条读 | 任何一条有问题 → 不是对的 |
| 8 | **诚实标记** | 回顾 1-7，标记任何 benefit of the doubt | 有模糊地带 → 结论降一档 |

### 修正循环

发现问题后不只会输出建议，而是：

```
灵魂发问发现问题 → 当场修复 → 重新验证 → 再次发问
     ↑                                    |
     └──────── 还有问题？继续修正 ────────┘
```

只有全部通过才允许声称完成。

### 三档判定

| 状态 | 含义 |
|------|------|
| 已完成 | 8 条全部无存疑，证据确凿 |
| 基本完成（有瑕疵） | 存在次要问题，但核心目标达成 |
| 未完成 | 核心目标未达成或发现实质性 Bug |

## 使用方式

**方式一：AGENTS.md 注入（推荐）**

在 `~/.config/opencode/AGENTS.md` 中加入：

```markdown
## 问心
每次声称完成或汇报前，必须调用 `skill({ name: "wen-xin" })` 执行验证。铁律，不可跳过。
```

每次会话自动生效，无需额外操作。

**方式二：触发词唤醒**

直接在 prompt 中以 `wen-xin` 开头：

> wen-xin，实现一个排序函数...

agent 会加载问心并在完成后输出验证报告。

两种方式不冲突：AGENTS.md 让问心成为默认习惯，触发词用于主动查看完整报告。

## 安装

```bash
git clone git@github.com:xp1001/wen-xin.git ~/.agents/skills/wen-xin
```

## 验证测试

用 3 个场景验证了问心效果：

| 场景 | 结果 |
|------|------|
| 完整任务（冒泡排序） | ✅ 正确判定「已完成」，8 条全部通过 |
| 遗漏日志（排序无日志） | ✅ 发问发现遗漏 → 自动补日志 → 再验证通过 |
| NaN Bug（去重函数） | ✅ 发问发现 Bug → 自动修复 → 测试通过 |
