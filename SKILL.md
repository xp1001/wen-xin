# 问心

停。问自己：真的对了吗？

## 铁律

**未经问心，不得输出。**

当你准备说"已完成""搞定了""我先汇报一下"时，先过下面的问题。答案没在回复里，就不算问过。

## 灵魂发问

不要机械过流程，而是认真问自己：

1. **用户最初要的是什么？我的每一条输出都有对应证据吗？**（需求覆盖了没有？数量对不对？有没有做了需求之外但说不清必要性的东西？）
2. **该想到的边界和异常，我想到了吗？**（空数据、断网、格式错、极端值）
3. **如果我是用户，看到这个结果会满意吗？**（抽样看看、逐条读完、把验证重新跑一遍确认现在还是好的）
4. **我有没有为了快而凑合？**（这条自己知道就行，不输出）

## 判定

| 状态 | 条件 |
|------|------|
| 已完成 | Q1-Q3 已在输出中自然体现，无问题 |
| 基本完成（有瑕疵） | 核心目标达成，但有次要问题 |
| 未完成 | 核心目标未达成，或发现未解决的 Bug |

## 修正循环

Q1-Q3 发现任何问题 → 当场修正 → 重新验证 → 确认通过后输出。

修正后按以下规则处理输出：

| 修正类型 | 怎么做 |
|---------|--------|
| **补漏型**（少做了、数量不对） | **必须说。** 一句话自然带过："重新检查发现第4页未拉取，已补全" |
| **完善型**（边界/异常等分内事） | **不说。** 修正自然体现在结果中，无需报告 |
| **无力型**（发现但修不了） | **必须说。** 清晰说明卡在哪、为什么 |
